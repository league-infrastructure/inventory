# Sprint 001 Use Cases

## SUC-001: Update README with Application Description

- **Actor**: Developer / new contributor arriving at the repository for the
  first time.
- **Preconditions**: The repository README.md exists but contains no
  description of what the application does or who it is for.
- **Main Flow**:
  1. Actor opens README.md.
  2. Actor reads a description section that names the application (League of
     Amazing Programmers Inventory System), states its purpose (tracking
     computing equipment and teaching materials across storage and teaching
     sites), and identifies the primary user roles (Instructor, Quartermaster,
     Admin).
  3. Actor understands what the application does without consulting any other
     file.
- **Postconditions**: README.md contains an accurate, readable description of
  the application.
- **Acceptance Criteria**:
  - [ ] README.md has a description that names the application.
  - [ ] The description explains the core purpose (equipment and material
        tracking across sites).
  - [ ] The description names the primary user roles.
  - [ ] No source code, configuration, or other documentation files are
        modified.
